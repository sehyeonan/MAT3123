# 기계학습과 응용 final project

한국어/영어권 관객의 **영화 리뷰 표현 차이**를 비교하기 위한 프로젝트입니다.  
동일 영화 **기생충(Parasite)**에 대해 **네이버(Naver) 리뷰(한국어)**와 **IMDb 리뷰(영어)**를 수집·전처리하고, 결과를 분석해보았습니다.

## 분석 관심사(가설/배경)
- 한국어 리뷰는 완곡·간접적 비판(체면/공손성 규범) 경향
- 영어 리뷰는 더 직접적·감정적으로 강한 평가 표현 경향
- 평점 분포도 영어권은 극단값(아주 높음/아주 낮음)이 더 많고, 한국어권은 중간값에 더 몰릴 가능성

## 디렉터리 개요
- `analysis/` : 워드클라우드/평점 비교 등 분석 스크립트 모음
- `data_pipeline/` : 전처리/임베딩 생성 등 파이프라인 코드
- `crawler/` : 크롤링 코드
- `output/` : 미리 수집·전처리된 결과물 (CSV, 임베딩 산출물 등)

## 데이터 수집(Data Collection)

### Naver (Korean)
- 네이버 영화 페이지에서 **“관람평” 탭**을 통해 리뷰를 수집합니다.
- 주요 추출 필드(리뷰 카드 단위)
  - 작성자ID, 작성일자, 리뷰 텍스트, 별점, 공감수/비공감수
- 저장 예시
  - `Parasite_naver_reviews.csv`
  - 컬럼 예: `author_id, rating, content, date, like, dislike`

### IMDb (English)
- IMDb 리뷰 페이지에서 **“See all”** → 스크롤 로딩 후 리뷰 카드를 수집합니다.
- 주요 추출 필드
  - author, title, rating(있으면), text
- 저장 예시
  - `Parasite_imdb_reviews.csv`
  - 컬럼 예: `author, title, rating, text`
 
  ## 전처리(Data Preprocessing)

### Korean (Naver)
- 토큰화 → 한글 토큰(길이 ≥ 2) 중심 필터링
- 커스텀 불용어(stopwords) 제거
- 리뷰 단위로 다시 결합한 클린 텍스트 생성

### English (IMDb)
- 소문자화 → 토큰화
- 패키지 내장 기본 불용어 + 영화 관련 커스텀 불용어(예: movie, director, spoiler 등) 제거
- 리뷰 단위로 다시 결합한 클린 텍스트 생성

## ANALYSIS RESULT

### 1) Data & Summary Statistics
- IMDb 리뷰: **2,997개**, Naver 리뷰: **1,090개** (둘 다 비교 분석에 충분한 표본)
- 전체 평점은 두 플랫폼 모두 **매우 긍정적**이며, 평균은 **약 8점대**
- 중앙값: **IMDb 9 / Naver 10** → Naver가 전반적으로 **조금 더 높은 점수 경향**
- 수집 기간: 두 데이터 모두 **2019-05-30 시작**, IMDb는 **2025-01-17까지**, Naver는 **2025-02-02까지**

### 2) Rating Distribution (플랫폼별 평점 분포)
- 두 플랫폼 모두 **오른쪽으로 치우친(right-skewed)** 분포
- **Naver**: 평점이 **10점에 강하게 집중**(최댓값에 매우 뾰족한 피크)
  - 동시에 **아주 낮은 평점(1점대)**도 IMDb보다 약간 더 존재 → **양극화(polarized) 가능성**
- **IMDb**: 평점이 **6~9 구간에 상대적으로 넓게 분포** → Naver 대비 **좀 더 “중간대” 평가**

### 3) Rating Trend Over Time (시간에 따른 평점 추이) + 데이터 수집 한계
- Naver 쪽은 특정 기간에 **표본이 매우 작아지는 구간**이 존재
- 크롤링 과정에서 **리뷰 수가 일정 임계치를 넘으면 과거 리뷰 일부가 비가시화/비접근**되는 문제가 있었음
- 따라서 전체 기간을 그대로 비교하는 것은 의미가 약해, **리뷰 볼륨이 충분하고 비교 가능한 기간 중심**으로 분석을 진행

### 4) Filtering Strategy (공정 비교를 위한 필터링)
- 두 플랫폼이 **겹치는 분기(quarters)** 중에서, **각 플랫폼 최소 20개 리뷰 이상**인 구간만 남겨 이후 분석을 수행
- 필터링 후에는 **요약 통계가 달라질 수 있음**을 전제로 비교
- 이후 타임라인을 여러 구간으로 나눠서 분석(예: **2020-04-01 전/후 평점 밀도 비교**)

### 5) Wordcloud & Keyword-based Analysis (단어 기반 분석)
- **Low-rating(1~6)** / **High-rating(7~10)** 구간으로 나눠 플랫폼별 워드클라우드 비교
- 특정 **키워드 선택 시**, 그 키워드를 포함한 리뷰들의 **평점 분포가 플랫폼별로 어떻게 다른지** 확인

### 6) Embeddings (리뷰 임베딩 시각화)
- 각 리뷰를 의미 기반의 **임베딩 벡터**로 변환한 뒤, 2D로 투영하여 구조를 시각화
- 투영 방법 설명
  - **PCA**: 전체 분산(글로벌 구조) 보존에 강점(선형)
  - **t-SNE**: 국소 이웃(로컬 구조) 보존에 강점(군집이 잘 갈라져 보일 수 있으나 군집 간 거리 해석은 어려움)
  - **UMAP**: 로컬 구조 보존 + 비교적 빠르고 안정적(레이아웃도 의미 있게 유지하려 함)
- 관찰/한계
  - 문장 임베딩 결과가 **한-영 리뷰를 완전히 섞어주진 못했고**, 언어별로 **분리된 클러스터**가 주로 형성
  - 다만 **각 언어 내부에서는 의미 있는 군집**이 나타나 “비슷한 의견끼리 모이는” 구조를 확인
  - 향후 개선: **한쪽 언어로 번역 후 임베딩 재계산** → 단일 언어 공간에서 플랫폼 간 유사도 포착 기대

### 7) Key Takeaways (요약 결론형 한 줄 정리)
- 같은 영화라도 IMDb와 Naver는 완전히 동일하게 평가하지는 않으며, 전체적으로는 둘 다 긍정적이지만 **기간을 나눠 보면 차이가 더 선명**
- **IMDb는 비교적 안정적**, 반면 **Naver는 후기로 갈수록 팬덤-driven & 초고평점화 경향**
- 고평점/저평점 리뷰가 다루는 핵심 주제(가족, 계급, 불평등, 한국사회 등)는 **거의 유사**하지만, **판단(찬양 vs 거부)**이 달라 극단적 반응을 만든다는 해석
